    Mkdocs-站点搭建教程¶

    参考教程：

    https://note.tonycrane.cc/cs/tools/mkdocs/

    bilibili站点优化教程

    https://squidfunk.github.io/mkdocs-material/

    环境搭建¶

    1.安装Mkdocs，在终端执行

    pip install mkdocs

    2.在你想要保存项目的文件夹内打开终端并执行：

    mkdocs new <your-project-name>
cd <your-project-name>

    3.此时你可以看见一个docs文件夹用于存放你的各种文件和资源、一个.yml文件用于配置网页的各种功能。docs文件夹内有一个index.md，这将在后续被渲染为网站首页，然后执行：

    mkdocs build --clean
mkdocs serve

    4.操作正确的话，可以在http://127.0.0.1:8000/看见渲染后的页面，且页面实时渲染，便于调整。

    5.想要添加其他页面只需要将.md文件放进docs文件夹即可

    网页托管¶

    为了能供他人访问，我们还需要将网页托管。以Github page为例：

    1.在github上新建一个repo

    2.在project所在的文件夹打开终端并执行

    git init
git remote add origin https://github.com/<your-username>/<your-repo>.git 
mkdocs gh-deploy    #后续修改文件夹后运行该指令即可更新托管网页

    3.稍作等待就可以在https://<your-user-name>.github.io/<your-repo>/打开自己的网页

    Customize¶

    目前，我们得到的Mkdocs网页并不好看，我们需要定制属于自己的网页，虽然这个过程比较费时费力，但却是一个一劳永逸的过程。我们可以使用主题Material，它内置了很多好用的插件，稍做配置即可实现很多有意思的功能，同时，也提供了丰富的可指定选项。运行下述指令以安装Material主题：

    pip install mkdocs-material -U

    之后我们需要在.yml文件中添加内容： 

    theme:
  name: "material"
    Material主题主题中有很多实用插件，下面介绍本人推荐的内容。

    设置导航栏¶

    当我们往docs文件夹内添加文件时，mkdocs会自动帮我们将文档添加进页面左侧导航栏中，为美化/自行设置导航栏，我们可以使用nav进行调整，如下： 

    nav:
  - 主页: index.md
  - Mkdocs搭建教程: mkdocs.md
    冒号前表示导航标题，后面则需要跟上对应.md文件。我们还可以在某一导航页面下继续添加子页面，如： 
    nav:
  - 学习笔记：
    - Mkdocs搭建： mkdocs.md
    - 电子电路基础： zjf.md

    明暗主题调节¶

    在theme下添加内容即可，其中toggle为调节按键，我们可以自行设置其logo与悬停文字；scheme为主题类型，slate为暗色主题，default为亮色主题；primary和accent分别表示点击按钮前后导航栏的颜色。 

    theme:
  palette:
    # 切换到自动模式
    - media: "(prefers-color-scheme)"
      toggle:
        icon: material/brightness-auto
        name: 切换到亮色模式

    # 切换到亮色
    - media: "(prefers-color-scheme: light)" 
      scheme: default
      primary: cyan
      accent: indigo
      toggle:
          icon: material/weather-night
          name: 切换到暗色模式

    # 切换到暗色
    - media: "(prefers-color-scheme: dark)"
      scheme: slate
      primary: indigo
      accent: cyan
      toggle:
          icon: material/weather-sunny
          name: 切换到自动模式

    搜索插件¶

    在.yml文件中添加下列内容即可，关于搜索还有许多拓展插件，感兴趣可以前往Material官网下载使用。 

    plugins:
    - search

    我的.yml文件¶

    site_name: C1fton's World
site_url: https://c1fton.github.io/Clifton-s-Mkdocs-Page/
repo_url: https://github.com/C1fton/Clifton-s-Mkdocs-Page.git

theme:
  name: "material"

  language: 'zh'      # 使用中文

  features: 
        - navigation.instant # 现在页面不会跳转,而是类似单页应用,搜索和各种跳转都是在当前页面完成,对美观有很大帮助
        - navigation.tabs # 页面上方的标签页
        - navigation.tracking # 页面滚动时，导航栏高亮当前页面
        - navigation.sections # 使导航栏分块
        - navigation.expand # 默认展开导航
        - navigation.prune # 只渲染当前页面的导航
        - toc.follow # 滚动的时候侧边栏自动跟随
        - navigation.top # 返回顶部按钮
        - navigation.indexes # 生成索引
        - search.suggest # 补全建议
        - search.highlight # 搜索结果高亮
        - search.share # 搜索结果分享
        - navigation.footer # 页脚提示下一章
        - content.code.copy # 代码段上的赋值按钮
        - content.code.annotate # 代码段上的注释按钮

  icon:
    logo: material/cat # 左上角的 logo

  palette:
    # 切换到自动模式
    - media: "(prefers-color-scheme)"
      toggle:
        icon: material/brightness-auto
        name: 切换到亮色模式

    # 切换到亮色
    - media: "(prefers-color-scheme: light)" 
      scheme: default
      primary: cyan
      accent: indigo
      toggle:
          icon: material/weather-night
          name: 切换到暗色模式

    # 切换到暗色
    - media: "(prefers-color-scheme: dark)"
      scheme: slate
      primary: indigo
      accent: cyan
      toggle:
          icon: material/weather-sunny
          name: 切换到自动模式

markdown_extensions:
    - admonition # 警告语法
    - def_list
    - footnotes
    - abbr
    - pymdownx.caret
    - pymdownx.mark
    - pymdownx.tilde
    - md_in_html
    - pymdownx.arithmatex: # latex支持
          generic: true
    - toc:
          permalink: true # 固定标题位置为当前位置
          toc_depth: 3 # 目录深度
    - pymdownx.highlight: # 代码块高亮
          anchor_linenums: true
          linenums: true # 显示行号
          use_pygments: true # 代码高亮
          pygments_lang_class: true
          linenums_style: pymdownx-inline # 行号样式,防止复制的时候复制行号
    - pymdownx.betterem # 强调美化,比如**text**会被美化
    - pymdownx.caret # 上标和下标
    - pymdownx.mark # 上标和下标
    - pymdownx.tilde # 上标和下标
    - pymdownx.keys # 显示按键组合
    - pymdownx.critic
    - pymdownx.details # 可以折叠的代码块 ??? note 可以让警告变成折叠的
    - pymdownx.inlinehilite
    - pymdownx.snippets
    - pymdownx.superfences
    - pymdownx.magiclink # 自动识别链接
    - pymdownx.smartsymbols # 智能符号
    - pymdownx.snippets # 代码段
    - pymdownx.tasklist:
          custom_checkbox: true # 自定义复选框
    - attr_list
    - pymdownx.emoji:
          emoji_index: !!python/name:material.extensions.emoji.twemoji
          emoji_generator: !!python/name:material.extensions.emoji.to_svg
    - pymdownx.superfences: # 代码块中支持Mermaid
          custom_fences: # 支持 Mermaid
              - name: mermaid
                class: mermaid
                format: !!python/name:pymdownx.superfences.fence_code_format
    - pymdownx.tabbed:
          alternate_style: true
          combine_header_slug: true
    - pymdownx.tasklist:
          custom_checkbox: true
          clickable_checkbox: true
    - meta # 支持Markdown文件上方自定义标题标签等
    - tables

plugins:
    - mkdocstrings:
          handlers:
              python:
                  paths: [src]
    - search # 搜索插件

extra:
    generator: false  #删除页脚显示“使用 MkDocs 材料制造”
    social:
        - icon: fontawesome/brands/github
          link: https://github.com/C1fton
          name: GitHub

nav:
  - 主页: index.md
  - Mkdocs搭建教程: mkdocs.md
